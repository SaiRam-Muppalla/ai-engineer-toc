## 📝 2. AI Engineer Resume Tips

### Resume Structure

```
Name | Location | Email | GitHub | LinkedIn | Portfolio

SUMMARY (2-3 lines)
"AI Engineer with experience building RAG systems, LLM applications, and
 deploying ML models to production. Built X projects, including..."

SKILLS
Languages: Python, SQL
ML/AI: PyTorch, HuggingFace, LangChain, Scikit-learn, NumPy, Pandas
MLOps: FastAPI, Docker, MLflow, GitHub Actions
Databases: PostgreSQL, Pinecone, FAISS, Redis
Cloud: AWS (EC2, S3), GCP, Modal

PROJECTS (most important section!)
Project Name | GitHub Link | Live Demo
- Built X that does Y, resulting in Z
- Used [tech] to achieve [measurable outcome]

EXPERIENCE (internships, freelance, open source)

EDUCATION
```

### Resume DOs and DON'Ts

| ✅ DO | ❌ DON'T |
|-------|---------|
| Quantify everything ("98% accuracy", "2x faster") | List technologies without context |
| Link to live demos and GitHub | Have broken links |
| Tailor to each job | Use a generic resume |
| 1 page for freshers | Make it 3+ pages |
| Use action verbs: Built, Deployed, Improved | Use passive voice |

## 💼 3. AI Engineer Interview Preparation

### Technical Interview Topics

#### Machine Learning
- [ ] Bias-variance tradeoff
- [ ] Regularization (L1 vs L2)
- [ ] Gradient descent variants (SGD, Adam, RMSprop)
- [ ] Overfitting and how to prevent it
- [ ] Feature selection techniques
- [ ] Handling imbalanced datasets
- [ ] Cross-validation strategies
- [ ] Ensemble methods (bagging, boosting)

#### Deep Learning
- [ ] Backpropagation math
- [ ] Vanishing/exploding gradients
- [ ] Batch normalization
- [ ] Attention mechanism and transformers
- [ ] CNN architecture design
- [ ] Transfer learning when and how

#### LLMs / NLP
- [ ] How transformers work (self-attention)
- [ ] BERT vs GPT — architectural differences
- [ ] How RAG works step by step
- [ ] Fine-tuning vs prompting tradeoffs
- [ ] Embedding models and vector similarity
- [ ] Hallucination and how to mitigate it
- [ ] Prompt injection and AI security

#### System Design for AI

| Component | Key Decisions |
|-----------|--------------|
| Data pipeline | Batch vs streaming, storage, preprocessing |
| Model training | Distributed training, checkpointing |
| Model serving | Latency vs throughput, batching, caching |
| Monitoring | Drift detection, alerting thresholds |
| Scalability | Horizontal scaling, load balancing |

**Practice System Design Questions:**
1. Design a real-time fraud detection system
2. Design a document search system for 10 million docs
3. Design a recommendation system for an e-commerce platform
4. Design a conversational AI customer support system
5. Design a model training and deployment pipeline

#### Coding Round (Python)
- Data manipulation with Pandas/NumPy
- Implement common algorithms (gradient descent, k-means, etc.)
- Debug ML code
- Write clean, tested Python code
